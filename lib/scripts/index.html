Requerimientos NO Funcionales

Código Requerimiento NO Funcional Descripción / Métricas
RNF01 Rendimiento UI Acciones críticas <3s (P95). Primer render <2s en equipos medios. Scroll fluido en listas largas. Animaciones <16ms por frame.
RNF02 Backend Scoring simulado <500ms (P95). Generación de cronograma <1s (P95). Consulta de expediente <200ms (P95). Tiempo de respuesta API <100ms (P95).
RNF03 Seguridad JWT de corta duración (15min). HTTPS obligatorio. Sin PII innecesaria en cliente. Validación estricta de entrada. Rate limiting en APIs críticas.
RNF04 Permisos Roles jerárquicos: Cliente (solo sus datos), Asesor (expedientes asignados + sucursal), Analista (todos los expedientes + decisiones), Admin (sistema completo + configuración).
RNF05 Idempotencia Desembolso y asientos por requestId. Operaciones seguras contra duplicación. Estado consistente ante fallos.
RNF06 Consistencia Transacciones atómicas en cambios de estado. Integridad referencial en relaciones. Estado consistente en reconexión.
RNF07 Observabilidad Logs estructurados con correlationId. Métricas: errores, latencia P95, throughput. Alertas en errores críticos. Dashboard básico de monitoreo.
RNF08 Calidad Crash-free ≥99% (medido por Firebase Crashlytics). Código cubierto ≥70% por tests. Linting estricto (sin warnings). Bundle móvil <50MB.
RNF09 Accesibilidad Texto escalable (14px mínimo). Labels claros y descriptivos. Navegación por teclado completa. Contraste ≥4.5:1 en elementos clave. Formatos regionales (Perú).
RNF10 Mantenibilidad Linter y formatter automáticos. CI con tests unitarios clave. Documentación de APIs y reglas de negocio. Código auto-documentado.
RNF11 Disponibilidad Horario de operación: 6:00-22:00 (hora Perú). Mantenimiento fuera de horario. Reinicio sin pérdida de datos. Recuperación automática de fallos.
RNF12 Backup Backup automático cada 24h. Retención mínima 7 días. Punto de recuperación <1h. Verificación de integridad.
RNF13 Telemetría UX Embudo de conversión: registro → pre-solicitud → aprobación → desembolso. Tasa de abandono por etapa. Tiempo promedio por flujo.
RNF14 Anti-spam CAPTCHA en endpoints públicos. Rate-limit por IP (máx. 3/hora). Validación estricta de entrada. Detección de bots básicos.
